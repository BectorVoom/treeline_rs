<objective>
Pull forward the MINIMAL GTIL surface each Phase-4 loader needs to verify 1e-5 (D-03/D-04). The current `treelite-gtil::predict` is scalar-only — flat `Vec<f32>` of `(num_row,1,1)` with identity/sigmoid (`lib.rs:143-239`). This plan widens output shaping to `(num_row, num_target, max_num_class)` routed by `target_id`/`class_id`, adds tree averaging (RF) and the f64 per-`(target,class)` base-score add, and ports four new postprocessors. Phase 5 later widens to the COMPLETE GTIL surface and does NOT backfill this.

Purpose: Every estimator-slice golden assert downstream needs this shaping + these postprocessors (RESEARCH Pitfall 6). It is the second Wave-1 gate, parallel to the f64 builder.
Output: multiclass/leaf-vector output shaping + averaging + base-score add; softmax / exponential_standard_ratio / exponential / logarithm_one_plus_exp postprocessors.
</objective>
