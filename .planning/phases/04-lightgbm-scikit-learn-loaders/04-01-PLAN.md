<objective>
Enable the `<f64,f64>` construction path that every Phase-4 loader depends on (D-05). Today `treelite-builder` only produces `Tree<f32>` / `ModelVariant::F32` (`lib.rs:135,558`); LightGBM and sklearn both require `<f64,f64>`. This plan adds an f64 builder mode and a bulk-trees→`Model` assembly, exercising the existing `ModelVariant::F64`/`Tree<f64>` core types end-to-end for the first time.

Purpose: This is the hard enabling gate (RESEARCH Pitfall 1 / Open Q2). Without it, routing the new loaders through the builder silently downcasts f64 thresholds/leaves and blows the 1e-5 bound.
Output: f64 `ModelBuilder` surface + `bulk_to_model` assembly; the f32 XGBoost path stays byte-identical.
</objective>
