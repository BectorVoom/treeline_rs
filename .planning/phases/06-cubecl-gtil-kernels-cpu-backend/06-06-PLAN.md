(CR-03 leaf-vector OOB). Each fix must ship with a test that would have caught the
original bug.

within 1e-5) for the FULL supported model set, not just XGBoost, and uphold the
T-06-09 no-OOB security contract on the cubecl CPU backend.

in-kernel comparison (CR-02), host-side leaf-vector span validation with a new
typed error variant (CR-03), the matrix sibling's provenance widened to keep
non-kLT (fallback) models honestly tagged, and a malformed-model regression test.
