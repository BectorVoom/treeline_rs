<objective>
Assemble the dense numerical-split traversal kernels across all four predict kinds (D-01) and wire the `predict_cpu` host launcher (validate → upload → select kernel by `Config.kind` → launch on `CpuRuntime` → read back) — with `<f32,f32>` AND `<f64,f64>` presets running in-kernel (D-05) and any model carrying a categorical split, or any sparse input, routed to the green `treelite_gtil` scalar fallback (D-02). Asserted within 1e-5 of the scalar reference across all kinds and both input dtypes.

Purpose: This is the real end-to-end-kernelized MVP slice — the dominant inference path (dense numerical, all 4 kinds, f32+f64, leaf-vector broadcast) running on cubecl, reproducing `predict_preset` line-by-line (serial tree-sum per row, RF averaging, f64 2D base-score add, leaf-vector broadcast). Separate kernels per predict kind (host-known from `Config.kind`) keep each kernel's control flow simple — the kind branch lives on the host (kernel selection), never in-kernel. The host launcher mirrors the scalar `predict`'s up-front shape validation before any device op. Categorical/sparse fall back whole-model (Open Q1 recommendation) so this slice's kernels stay purely numerical; the fallback is honest and recorded as provenance in plan 05.
Output: A green `predict_kinds` test asserting all 4 kinds + leaf-vector broadcast within 1e-5 of `treelite_gtil::predict` on both input dtypes; `predict_cpu` is the real host launcher (no longer the plan-01 stub).
</objective>
