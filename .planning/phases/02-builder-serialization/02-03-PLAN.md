<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted bytes → `deserialize` | A v5 blob may be hostile: attacker-controlled u64 array/string count prefixes, `num_tree`/`num_nodes`, version header, and total length. THE primary threat surface of this phase. |
| `&Model` → PyBuffer frames | Read-only borrow; no untrusted input, but frame lifetime must not outlive the model (compiler-enforced via `'a`). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-S01 | Denial of Service | `deserialize` array/string read | mitigate | Before allocating, verify `count * size_of::<T>() <= remaining_bytes` (+ absolute cap) → typed `CountExceedsBuffer`; never `Vec::with_capacity` an untrusted u64 (RESEARCH § Security). |
| T-02-S02 | Tampering / Info disclosure | `Reader` cursor | mitigate | Every read checks `offset + n <= buf.len()`; short read → typed `TruncatedStream`; no `unsafe` slicing in the read path (safe Rust slicing converted to `Result`, not allowed to panic). |
| T-02-S03 | Tampering | version gate | mitigate | D-03: `major_ver != 4` → typed `UnsupportedVersion` early; the V3 parse branch is NOT ported, so a 3.9 header cannot mis-route. |
| T-02-S04 | Denial of Service | opt-field skip loop | mitigate | `num_opt_field_*` counts come from the stream; bound them (we always write 0; reject absurd counts on read) so the skip loop cannot be driven to amplify allocation. |
| T-02-S05 | Tampering | NaN/inf float columns | accept | Floats are raw IEEE-754 bits (`to_le_bytes`); NaN/inf round-trip bit-exact by design — round-trip identity is asserted at the BYTE level, never float-`==` (Pitfall 4). No security risk; documented to prevent a false "corruption" reading. |
| T-02-SC | Tampering | crate manifest deps | mitigate | No new third-party packages; package-legitimacy gate N/A (RESEARCH § Package Legitimacy Audit). |
</threat_model>
