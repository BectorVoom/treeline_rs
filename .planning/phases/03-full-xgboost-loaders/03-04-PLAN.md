## Phase Goal

**As a** model engineer, **I want to** load any of XGBoost's three on-disk formats (JSON, UBJSON, legacy binary) into treelite-rs with auto-detection, **so that** the same logical model predicts within 1e-5 of upstream Treelite and serializes byte-identically regardless of which format it came from.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-L01 | Denial of Service | oversized `u64` objective/booster name length or leaf-vector len → giant allocation (OOM) | mitigate | Cursor `buf.get(pos..pos+len)` returns `Option`; len validated against remaining buffer before reading → `XgbError::Legacy`, never huge alloc |
| T-03-L02 | Denial of Service | huge/negative `num_nodes` or `num_trees` → OOM / capacity overflow | mitigate | `num_nodes > 0` assert (ports TREELITE_CHECK_GT); fixed-stride reads (`num_nodes × 20`, `× 16`) bounds-checked against remaining bytes via the `Option` cursor before iterating |
| T-03-L03 | Denial of Service | truncated file mid-struct | mitigate | Every cursor read is `Option`-checked → `XgbError::Legacy` truncation, never OOB/panic (asserted in legacy.rs test) |
| T-03-L04 | Tampering | native-endian/padding mismatch via struct transmute | mitigate | D-08: field-by-field `from_le_bytes`; `transmute`/`bytemuck::cast` forbidden — enforced by a grep gate in acceptance criteria |
| T-03-L05 | Tampering | `sindex` mis-unpack → wrong split index / default direction | mitigate | Exact bit ops `& 0x7FFFFFFF` / `>>31` (Pitfall 6); the 1e-5 predict + byte-fidelity asserts catch any divergence |
| T-03-L06 | Spoofing | base64 (`bs64`) framing | mitigate | Explicitly rejected → `XgbError::Legacy` (ports xgboost_legacy.cc:331) |
| T-03-SC | Tampering | npm/pip installs | accept | No new crates; zero-dependency `from_le_bytes` cursor (D-07 rejects byteorder/binread) |
