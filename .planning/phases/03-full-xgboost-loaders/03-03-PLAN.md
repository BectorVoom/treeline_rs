## Phase Goal

**As a** model engineer, **I want to** load any of XGBoost's three on-disk formats (JSON, UBJSON, legacy binary) into treelite-rs with auto-detection, **so that** the same logical model predicts within 1e-5 of upstream Treelite and serializes byte-identically regardless of which format it came from.

## Artifacts this phase produces (Plan 03 — UBJSON + detect slice)

New module files: `crates/treelite-xgboost/src/ubjson.rs`, `crates/treelite-xgboost/src/detect.rs`. New test files: `crates/treelite-xgboost/tests/ubjson.rs`, `crates/treelite-xgboost/tests/detect.rs`.

New public + crate symbols: `pub fn detect_xgboost_format(&[u8]) -> &'static str`; `pub fn load_xgboost_ubjson(&[u8]) -> Result<Model, XgbError>`; `pub(crate) fn decode_ubjson(&[u8]) -> Result<serde_json::Value, XgbError>`; new `XgbError::Ubjson` variant; `mod ubjson;` / `mod detect;` module declarations + re-exports in `lib.rs`.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-U01 | Denial of Service | UBJSON `$`/`#` declared container count larger than the remaining stream → giant pre-allocation (OOM) | mitigate | Validate the decoded `#`count against `bytes.len() - pos` BEFORE `Vec::with_capacity`/pre-alloc; over-large → `XgbError::Ubjson` |
| T-03-U02 | Denial of Service | truncated UBJSON mid-tag / mid-string-length | mitigate | Cursor uses `bytes.get(pos..pos+n)` → `Option`; every read fallible → typed `XgbError::Ubjson`, never OOB/panic |
| T-03-U03 | Tampering | non-finite float silently lost (→ Value::Null) causing wrong Model | mitigate | Non-finite `d`/`D` emit sentinel `Value::String` routed through `de_f32` (Pitfall 5) — value preserved, parity with JSON; the D-10 byte-fidelity assert catches any desync (A4) |
| T-03-U04 | Tampering | type-tag confusion (untyped decode of a `$`-typed container) desyncs the stream | mitigate | Explicit `$`/`#` fast-path handling (Pitfall 4); the same-Model-as-JSON assert detects any desync |
| T-03-SC | Tampering | npm/pip installs | accept | No new crates; zero-dependency hand-rolled decoder (D-03 rejects serde_ubjson/ubjson crates) |
</threat_model>
