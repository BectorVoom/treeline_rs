## Phase Goal

**As a** model engineer, **I want to** load any of XGBoost's three on-disk formats (JSON, UBJSON, legacy binary) into treelite-rs with auto-detection, **so that** the same logical model predicts within 1e-5 of upstream Treelite and serializes byte-identically regardless of which format it came from.

## Artifacts this phase produces (Plan 02 — JSON slice)

New module file: `crates/treelite-xgboost/src/json.rs`. New test files: `crates/treelite-xgboost/tests/json.rs`, `crates/treelite-xgboost/tests/nan_inf.rs`.

New/changed public + crate symbols: `treelite_xgboost::json` module; `pub(crate) fn replace_nonfinite(&str) -> String`; `fn de_f32` / `fn de_vec_f32` serde adapters; `pub(crate) fn build_model_from_parsed(XgbModelJson) -> Result<Model, XgbError>` (shared convergence path); `pub fn parse_base_score(&str, usize, &str, bool) -> Result<Vec<f64>, XgbError>` in `objective.rs`; widened `XgbModelJson`/`Learner`/`LearnerModelParam`/`GradientBooster`/`BoosterModel`/`RegTreeJson`/`TreeParam` struct fields (recognized key set: `loss_changes`, `sum_hessian`, `base_weights`, `leaf_weights`, `categories_segments`/`categories_sizes`/`categories_nodes`/`categories`, `boost_from_average`, `tree_info`, `cats`, `weight_drop`, `num_feature`, `size_leaf_vector`); new `XgbError` variant for vector base_score shape (e.g. `BaseScoreShape`). The `attributes: None` change to the loader metadata.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-V01 | Tampering | per-tree parallel arrays (`split_conditions`, `sum_hessian`, `loss_changes`) length != num_nodes | mitigate | `check_dim(tree, field, num_nodes, got)` runs for EVERY parsed array BEFORE any builder emission (ports delegated_handler.cc:411-432) → typed `XgbError::DimensionMismatch`, never OOB index |
| T-03-V02 | Tampering | NaN/Inf injected into float arrays to crash the lexer | mitigate | D-02 sentinel mechanism: non-finite values are EXPECTED and recovered via `de_f32`, not crashes; pre-lexer is string-safe (in-string state tracked) so attribute strings cannot be corrupted |
| T-03-V03 | Denial of Service | negative/huge `num_feature`/`num_class`/`num_target` → capacity-overflow allocation | mitigate | existing `require_non_negative` runs before any `vec![..; n as usize]`; vector base_score length validated against `num_target*num_class` |
| T-03-V04 | Tampering | vector base_score string with wrong element count | mitigate | `parse_base_score` asserts `len == num_target*num_class` → typed error, no silent truncation |
| T-03-SC | Tampering | npm/pip installs | accept | No new crates added; runtime crate set unchanged from Phase 2 (already vetted) |
</threat_model>
