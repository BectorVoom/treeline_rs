## Phase Goal

**As a** model engineer, **I want to** load any of XGBoost's three on-disk formats (JSON, UBJSON, legacy binary) into treelite-rs with auto-detection, **so that** the same logical model predicts within 1e-5 of upstream Treelite and serializes byte-identically regardless of which format it came from.

## Artifacts this phase produces (Plan 01 — Wave 0)

New files: `fixtures/generate_xgb_3format.py`, `fixtures/xgb_3format.json`, `fixtures/xgb_3format.ubj`, `fixtures/xgb_3format.model`, `fixtures/xgb_3format.golden.json`, `fixtures/golden_v5_3format.bin`, `fixtures/xgb_3format.manifest.json`, `crates/treelite-harness/tests/three_format_equivalence.rs`.

This plan creates fixtures/goldens/manifest and a RED test scaffold only — no new Rust symbols in the loader crate. Downstream loader/decoder symbols are produced in plans 03-02..03-04 (each lists its own).
