<objective>
Widen `treelite-gtil` in place to the typed `Config`/`PredictKind` entry surface (D-06), the public `Shape`/`output_shape` descriptor (D-07), and the `O`-generic input/output element path (D-05) — so `predict` works over both f32 and f64 input matrices for either model preset, dispatches `Default` vs `Raw`, and the binary path stays byte-identical.

Purpose: This is the foundation slice the remaining GTIL plans build on. It establishes the orthogonal output-element trait (the central 1e-5 correctness pattern: output type == InputT, NOT the leaf type) and the per-kind shape contract Phase-8 numpy reshapes against. It does NOT add sparse or the two new kinds (Plan 04) or the new postprocessors/categorical guard (Plan 03).
Output: `config.rs`, `shape.rs` (new), an `O`-generic `predict` + Config dispatch in `lib.rs` with the internal `Shape<'m>` renamed `OutputLayout`.
</objective>

## Phase Goal

**As a** treelite-rs maintainer validating the inference engine, **I want to** assert Rust GTIL predictions within 1e-5 of frozen upstream-Treelite goldens across the full model × preset × input-dtype × kind × {dense,sparse} × seed matrix, **so that** every later compute backend has a trusted 1e-5 measurement instrument.
