<objective>
Build the exhaustive equivalence-matrix runner that turns the Plan-01 RED `gtil_matrix.rs` GREEN: iterate the committed `fixtures/gtil/` cross-product, assert each Rust prediction within 1e-5 of its upstream golden and report max observed `|delta|` (EQV-03, EQV-04), dispatch the f32-vs-f64 input path per the fixture `input_dtype` (no pre-cast, D-05), assert Rust-dense == Rust-sparse parity per pair (D-04), extend the `Manifest` with a `backend` field + provenance (D-09), and add the minimal backend-parameterized seam (D-11) so Phase 6 plugs a cubecl `R: Runtime` in by registration, not refactor.

Purpose: This is the 1e-5 measurement instrument every later phase trusts. It generalizes the existing per-loader golden-assert pattern into one uniform, backend-ready matrix harness. NO cubecl code this phase — only the seam design (one `Backend::ScalarCpu` variant + a dual input-dtype predict-fn registry).
Output: extended `Manifest` (`manifest.rs`), `Backend`/`PredictFn` seam (f32 + f64 input entry points) + runner helpers (`lib.rs`), GREEN `gtil_matrix.rs`.
</objective>
