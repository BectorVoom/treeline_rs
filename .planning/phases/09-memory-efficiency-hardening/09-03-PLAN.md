<objective>
MEM-01 (D-01: push Pod recasting through the existing `tree_buf.rs` `as_bytes` seam wherever
layout allows, bounded by the byte-identical golden): replace the hand-rolled
`unsafe { from_raw_parts }` in `le_bytes_of` with the existing, tested `bytemuck::cast_slice`
seam (the exact form already in `tree_buf.rs:96-105`), tightening the bound `T: Copy` →
`T: bytemuck::Pod` and documenting the little-endian-host-only assumption (D-02) at the site.
All call sites pass `i32`/`f64` columns that are already `Pod`, so the change is mechanical and
the emitted bytes are identical on the LE manifest host (D-03). The recast is restricted to the
safe `&[T]→&[u8]` EMIT direction only — the untrusted deserialize read path keeps its element-wise
bounds-checked decode (security), and the enum/bool tree columns that define their own wire byte
are NOT recast (D-01's "where layout allows" boundary).

Purpose: closes MEM-01 by routing the serializer's POD-column emit through the validated bytemuck
seam instead of a raw transmute, removing an `unsafe` block while staying byte-identical. The golden
byte-compare (D-03) and the 1e-5 harness (D-11) are the gates.
Output: one-function change in serialize/mod.rs, golden + equivalence green.
</objective>
