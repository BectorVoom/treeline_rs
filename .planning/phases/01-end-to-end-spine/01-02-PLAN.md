## Phase Goal

**As a** treelite-rs consumer, **I want to** load one XGBoost-JSON model into the core `Model` representation, **so that** the loaded model is ready for an equivalence-verified predict — proving the loader leg of the walking-skeleton pipeline.

<objective>
Implement `treelite-xgboost::load_xgboost_json` — the minimal XGBoost-JSON loader that parses the committed fixture into a `treelite_core::Model` (always the F32 variant), porting the objective→postprocessor map and the version-gated f64 `base_score`→margin transform verbatim from upstream.

Purpose: Wave 2 vertical-slice step — turns the committed fixture into a live `Model`. After this task a consumer can call `load_xgboost_json` on the fixture and get a `Model` with `postprocessor == "sigmoid"` and a correctly margin-transformed `base_scores`. This plan produces the live f64 margin-transformed value of CORE-04's `base_scores` field (which 01-01 only stubs structurally). Implements the XGB-01 subset (one JSON model), the objective map, the base_score transform, and ERR-01 typed errors.

Output: A working `treelite-xgboost` crate with a typed error surface and load + error tests.
</objective>
