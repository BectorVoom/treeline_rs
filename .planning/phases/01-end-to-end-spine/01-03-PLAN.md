(2) For each row, slice `row = &data[r*num_feature .. (r+1)*num_feature]`; iterate trees **serial in tree_id order** (no parallelism across the tree axis — float add is non-associative, GTIL-08); for each tree call `evaluate_tree` to get the leaf node id, then `output[r] += tree.leaf_value(leaf) as f32` (mirror `static_cast<InputT>` at predict.cc:228). (5) Apply the postprocessor selected by `model.postprocessor`: `"identity"` => `identity`, `"sigmoid"` => `sigmoid(model.sigmoid_alpha, output[r])`; any other name => `Err(GtilError::UnsupportedPostprocessor)`. Implement `fn evaluate_tree<T: Copy + Into<f64>>(tree: &Tree<T>, row: &[f32]) -> Result<usize, GtilError>`: start `nid = 0`; while `!tree.is_leaf(nid)`: read `fi = tree.split_index(nid)`; bounds-check `fi` against `row.len()` returning `Err(GtilError::FeatureIndexOutOfBounds)` on overflow (ASVS V5 — no OOB panic); `fvalue = row[fi as usize]`; if `fvalue.is_nan()` => `nid = tree.default_child(nid) as usize` (missing-value routing via default direction); else `nid = next_node(fvalue, tree.threshold(nid), tree.comparison_op(nid), tree.left_child(nid), tree.right_child(nid)) as usize`. Implement `fn next_node(fvalue: f32, threshold, op: Operator, left: i32, right: i32) -> i32` with the comparison switch from predict.cc:99-124 (kLT: `fvalue < threshold`, kLE: `<=`, kEQ: `==`, kGT: `>`, kGE: `>=`), returning `left` if the condition holds else `right`. XGBoost always uses `kLT`. Match on `ModelVariant::F32`/`F64` to dispatch over the typed trees. Do NOT introduce a Predictor/backend trait (D-08) — a plain `fn predict` is correct for Phase 1.
