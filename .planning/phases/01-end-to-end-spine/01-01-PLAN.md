## Phase Goal

**As a** treelite-rs consumer (and the project itself), **I want to** load one XGBoost-JSON model, predict it, and verify the output matches upstream Treelite within 1e-5, **so that** the core fidelity value is proven end-to-end on day one before any layer is built out to full breadth.

<objective>
Stand up the Cargo workspace and the `treelite-core` foundation layer — the four upstream enums, the `TreeBuf<T>` SoA storage primitive, the `Tree<T>` struct-of-arrays node columns, and the two-variant `Model` enum with full header metadata — plus the committed fixture + frozen golden artifact that defines the expected end-to-end behavior the rest of the phase is built to satisfy.

Purpose: This is Wave 1 of the walking skeleton. Everything downstream (loader, predict, harness) borrows `treelite-core`. The golden artifact is the "failing test first" of MVP mode: it commits the expected output before any Rust code computes it. Implements D-01 (spine-only workspace), D-02 (crate layout), D-03 (pinned deps), D-04/D-05/D-05a (hand-crafted fixture), D-06/D-07 (golden + manifest), CORE-01..04, ENUM-01, ERR-01, FND-01/FND-02.

Output: A buildable virtual workspace with four crate members (three of them minimal stubs wired for later waves), a fully-implemented `treelite-core`, the committed XGBoost-JSON fixture, and the frozen `golden.json`.
</objective>
