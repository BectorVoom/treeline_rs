## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| golden.json file → harness | Trusted committed fixture, but malformed JSON should surface as anyhow context, not a raw panic |
| running environment → manifest check | Environment drift (different glibc/OS than capture) is a diagnosability concern, not a security boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information/DoS | malformed or missing golden.json | mitigate | `load_golden` uses `.context(...)` so a parse/read failure surfaces a clear anyhow error chain rather than an opaque panic (ERR-02) |
| T-04-02 | Repudiation/Correctness | undiagnosed libm/environment divergence | accept | `check_manifest` warns (does not fail) on OS/arch/glibc drift; D-07 manifest makes a 1e-5 failure on a different distro immediately diagnosable |
| T-04-SC | Tampering | crates.io installs (anyhow, approx, serde, serde_json) | accept | Pinned exact versions from RESEARCH §Package Legitimacy Audit (slopcheck [OK]); `approx` pinned `0.5.1` not the `0.6.0-rc2` pre-release (FND-02) |
