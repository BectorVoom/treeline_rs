<objective>
Generalize the cubecl host launcher in `crates/treelite-cubecl/src/lib.rs` from the hardcoded `CpuRuntime` to a generic `R: Runtime` ŌĆö the single central code change of Phase 7. The kernels (`kernels::*::launch::<F, T, R>`) and `upload_forest::<R, F>` are ALREADY generic; this plan threads `R` through the four launcher fns and their three `launch::<F, T, CpuRuntime>` call sites, replaces the `CpuRuntime::client(...)` line with `device::client::<R>()?` (the Plan-01 constructor), and keeps a thin `predict_cpu::<F>` shim so the Phase-6 cubecl-cpu surface (`cubecl_cpu_case()`, `gtil_matrix_cubecl.rs`) compiles and passes byte-identical.

Purpose: Turn `CpuRuntime` into `R` at exactly one file so the same kernels run on ROCm/CUDA/wgpu (Plan 03 just registers them). DO NOT touch `kernels/*` or `upload.rs` ŌĆö if this plan edits them, that is the documented "smell."
Output: A runtime-generic `predict::<R, F>` + `predict_cpu` shim + `R`-threaded `launch_*`/`run_*` fns, with the CPU 1e-5 gate untouched.
</objective>
