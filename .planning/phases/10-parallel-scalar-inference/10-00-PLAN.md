| `&Model` shared across rayon workers | concurrent read-only access to `TreeBuf::Borrowed { *const T }` foreign buffers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Denial of Service | scoped thread pool sized by `Config.nthread` | mitigate | This plan adds `GtilError::ThreadPool(String)` so a bounded-pool build failure is a typed error (Wave 1 maps `ThreadPoolBuilder::build()` Err to it); `nthread > 0` is the explicit caller bound, `≤0` is core-capped. No `nthread`-driven heap growth. |
| T-10-03 | Tampering / Info Disclosure | `unsafe impl Sync for Model` | mitigate | The Sync impl is sound ONLY because predict is read-only (`&Model`, no interior mutability, `stage_serialization_fields` takes `&mut self`); the documented SAFETY argument is the control. `requires_sync::<Model>()` pins the contract; concurrent reads of an immutable `*const T` slice are race-free. Sync-only (not Send) keeps the contract minimal (A4). |
| T-10-SC | Tampering | crates.io install of `rayon` | mitigate | `rayon` audited in 10-RESEARCH §Package Legitimacy Audit: registry crates.io, ~10yr, >300M downloads, github.com/rayon-rs/rayon, slopcheck 0.6.1 `[OK]`, Approved (not `[ASSUMED]`/`[SUS]`/`[SLOP]`) — no human legitimacy checkpoint required. |
