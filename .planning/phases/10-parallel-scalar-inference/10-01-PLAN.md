<objective>
Row-parallelize the scalar GTIL engine. Convert the four serial outer row loops in `treelite-gtil/src/lib.rs` to rayon `par_chunks_mut` + `map_init` (per-worker scratch), thread `Config.nthread` through to a scoped thread pool, keep the inner per-row `tree_id` loop serial (GTIL-08), and prove parallel == serial within 1e-5 plus determinism, nthread honoring, and the Python kwarg end-to-end.

Purpose: LightGBM `kLE`, categorical, non-`kLT`, and all-sparse models stop running on one core (measured 99% → prototype 3.0–4.6×), with zero numerical regression. Closes PAR-01, PAR-02, PAR-04.
Output: parallel dense + sparse predict; honored nthread; green determinism/nthread/utilization/pytest gates; the frozen `gtil_matrix` 1e-5 goldens still pass against the parallel output.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller / Python → `Config.nthread` | untrusted integer now sizes a scoped rayon thread pool |
| `&Model` shared across rayon workers | concurrent read-only access to `TreeBuf::Borrowed { *const T }` foreign buffers (Sync from Wave 0) |
| CSR input → parallel row body | untrusted `col_ind`/`row_ptr` validated once, then read per row in parallel |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Denial of Service | `nthread`-sized scoped pool in predict_preset / leaf / score | mitigate | `nthread > 0` builds a pool bounded to exactly that many workers via `ThreadPoolBuilder::num_threads(n)`, `build()` failure → `GtilError::ThreadPool` (typed, no panic); `nthread <= 0` uses the core-capped global pool. NEVER `build_global` (once-only global mutation). Per-worker scratch is `num_feature`-sized, not `nthread`-driven heap growth. |
| T-10-02 | Denial of Service | CSR validation under parallelism | mitigate | `predict_sparse` validates the whole CSR ONCE up front (`csr.validate(num_row, num_feature)` at lib.rs:969) BEFORE the parallel section; validation is NEVER moved into the per-row closure (no per-row re-validation / DoS amplification). The dense buffer-length check stays up front too. |
| T-10-03 | Tampering / Info Disclosure | concurrent reads of the shared `Model` foreign buffer | mitigate | Relies on the Wave-0 `unsafe impl Sync for Model` soundness (read-only `&Model`, no interior mutability); workers share `&Model`, never mutate; `par_chunks_mut` yields statically-disjoint output slices so the borrow checker proves non-overlapping writes (no manual unsafe indexing). Determinism test proves no run-to-run reordering. |
| T-10-04 | Denial of Service | panic crossing the rayon boundary | mitigate | Prefer `Result` propagation: each closure returns `Result<(), GtilError>` and `.collect::<Result>()?` short-circuits on the first Err (ERR-01 typed-error parity). The existing treelite-py `guard_assert` catch_unwind still traps any residual worker panic into a `TreeliteError`. |
</threat_model>
